# Meal History

Running log of dinners. Source of truth for the no-repeat rule and mainstay identification.

## Format

Each entry is a level-3 heading with the date in YYYY-MM-DD format, followed by:

- **Meal:** the dish name
- **Description:** one-line description (mirrored from `current-week.md`)
- **Active time:** `X min  |  **Total time:** Y min` (mirrored from `current-week.md`)
- **Protein:** chicken / beef / pork / seafood / pasta / vegetarian / etc.
- **Recipe:** URL, or `no link — see description`
- **Rating:** `hit` / `miss` / `mixed` / blank (not yet rated)
- **Notes:** modifications, kid reactions, what to change next time

The first five fields are written by the weekly workflow and mirror what landed in `current-week.md` for the same day. **Rating** and **Notes** are for Ryan/Gina to fill in during or after the week.
